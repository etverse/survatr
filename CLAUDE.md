# survatr

Causal survival analysis via pooled-logistic hazard g-computation, IPW, and
iterated conditional expectation (ICE) on person-period data. Extends the
[`causatr`](https://github.com/etverse/causatr) engine to time-to-event
outcomes with delta-method sandwich variance on cumulative-product survival
curves, risk / RMST contrasts, competing-risks cause-specific hazards, and
empirical bootstrap. Part of the [etverse](https://github.com/etverse)
ecosystem.

## S3 classes

- `survatr_fit` — returned by the main fit function. Holds hazard model(s),
  person-period data, track (A / B), track-specific metadata.
- `survatr_result` — returned by `contrast()`. Holds time-indexed `estimates`
  and `contrasts` `data.table`s, `time_grid`, and the full vcov as an
  `n × |t-grid|` IF matrix → `|t-grid| × |t-grid|` covariance.
- `survatr_diag` — returned by `diagnose()`. Holds per-period positivity,
  per-period balance, competing-risks decomposition summaries.

## Two-step API

```r
# Step 1: fit the hazard model once
fit <- surv_fit(
  data,
  outcome = "Y",
  treatment = "A",
  confounders = ~ L1 + L2,
  id = "id",
  time = "time",
  censoring = "C",
  time_formula = ~ ns(time, df = 4),  # alpha(t) baseline hazard
  estimator = "gcomp",                 # or "ipw" / "ice"
  model_fn = stats::glm                # defaults to glm with binomial()
)

# Step 2: contrast many interventions, curve-shaped result
result <- contrast(
  fit,
  interventions = list(a1 = causatr::static(1), a0 = causatr::static(0)),
  times = seq(0, 120, by = 12),        # time grid
  type = "risk_difference",            # or "risk_ratio" / "rmst_difference"
  ci_method = "sandwich"               # or "bootstrap"
)
```

## Scope

survatr is a **causal survival analysis package** for time-to-event outcomes,
built on top of causatr. It owns: pooled-logistic hazard g-computation (Track
A), iterated-conditional-expectation hazards for longitudinal survival
(Track B), IPW weighted hazard MSM, cause-specific hazards + CIF for
competing risks, and sandwich variance via delta-method propagation through
the cumulative product. It is NOT: a Cox-PH modelling package (use
`survival`), a TMLE/ML survival package (use `lmtp`), a forward-simulation
g-formula for survival (use `gfoRmula`), or a Fine–Gray subdistribution
hazards package.

### Why a separate package (see handoff §1)

1. Person-period (long) table is the natural runtime, not a scalar row per
   individual.
2. The estimand is a **curve** (survival, CIF, RMST), not a scalar — the
   `causatr_result` S3 shape does not fit.
3. Competing risks via cause-specific hazards is first-class here.
4. Pooled logistic / discrete-time hazard is the outcome model, not a GLM on
   scalar Y.
5. Sandwich variance propagates through the cumulative product via a
   cross-time delta chain — a different aggregation than `causatr`'s
   `variance_if()`.
6. Matching + survival is rejected (use `survival::coxph(..., weights =
   match_weights, cluster = subclass)` directly on `MatchIt` output).

### Why ICE, not forward simulation (inherited from causatr)

Hernán & Robins Ch. 21 describes two longitudinal g-formula approaches.
Forward simulation (`gfoRmula`-style) needs models for every time-varying
covariate + outcome and bootstrap-only inference. ICE (Zivich et al. 2024)
models the outcome only at each time, iterates backward with a pseudo-
outcome, and admits stacked EE sandwich. Here the pseudo-outcome is the
**survival tail** under the intervention; per-step link is binomial at K,
quasibinomial at k < K.

## Key design decisions

- **Pooled logistic hazard, not Cox.** Discrete-time model on person-period
  rows. For small per-interval hazards (< 0.1) the two agree closely
  (Hernán & Robins TP 17.1; D'Agostino et al. 1990). Chosen for:
  sandwich-friendly M-estimator form, clean IF decomposition, alignment
  with Ch. 17 of Hernán & Robins.
- **Cumulative product at the individual level.** `S^a_i(t) = ∏_{k ≤ t}
  (1 − h^a_{i,k})`, then `S^a(t) = (1/n) Σ_i S^a_i(t)`. Never average
  hazards before cumulating (Jensen: the product is nonlinear).
- **Delta-method cross-time IF aggregation.** Per-individual IF is an
  `n × |t-grid|` matrix; full covariance = `crossprod(IF) / n^2`; pointwise
  SE = diagonal; RMST SE = quadratic form with trapezoidal weights.
  Pointwise bands only in v1.
- **causatr is the engine.** All IF primitives (`prepare_model_if()`,
  `apply_model_correction()`, `vcov_from_if()`, numeric fallback),
  intervention constructors (`static()`, `shift()`, ...), and the
  `to_person_period()` reshape live in causatr. survatr layers the
  cross-time aggregation and survival-curve estimand shape on top.
- **`model_fn`, not hardcoded link.** User passes `stats::glm`,
  `mgcv::gam`, etc. Defaults to `stats::glm` with `family = binomial()`
  (or `quasibinomial()` for weighted fits).
- **`estimator`, not `method`.** Same rule as causatr — `method` is
  reserved for `...`-forwarding (though matching is rejected here).
- **Two-step API.** Fit the hazard model once; contrast many interventions.
  Survival curves live in the `contrast()` return, not the fit.
- **Curve-shaped `survatr_result`.** Time-indexed `data.table`s for
  `estimates` and `contrasts`. S3 methods (`print`, `plot`, `tidy`)
  dispatch on the time-indexed shape. `forrest`-style forest plots at a
  user-chosen reference time `t*`.
- **Matching + survival rejected.** `survival::coxph(..., weights =
  match_weights, cluster = subclass)` on the `MatchIt` output is the
  correct tool and lives outside survatr. Hard-abort with error class
  `survatr_matching_rejected`.
- **Competing risks: cause-specific hazards + CIF only.** Fine–Gray /
  subdistribution hazards out of scope (different estimand, different
  data structure). Document explicitly.
- **Reserved column prefix `.survatr_*`.** `.survatr_prev_event`,
  `.survatr_prev_cens` for within-id lagged cumsum columns used by the
  risk-set construction. `check_reserved_cols()` guards against user
  collisions upfront.
